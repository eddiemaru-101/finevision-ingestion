# 실행 방법 및 의존성

## 요구 사항

- Python 3.11 이상
- [uv](https://github.com/astral-sh/uv) 패키지 매니저

---

## 의존성 패키지

| 패키지 | 버전 | 용도 |
|--------|------|------|
| `datasets` | 4.8.4 | HuggingFace 데이터셋 스트리밍 로드 |
| `pillow` | 12.1.1 | 이미지 저장 (PIL.Image.save) |
| `tqdm` | 4.67.3 | 진행 상황 출력 |
| `pyarrow` | 23.0.1 | Parquet 파일 쓰기 (datasets 의존성으로 자동 설치) |

> `pyarrow`는 `datasets` 설치 시 자동으로 함께 설치됩니다. `pyproject.toml`에 별도 선언 불필요.

---

## 설치

```bash
# 저장소 클론 후 프로젝트 디렉토리로 이동
cd finevision-ingestion

# 의존성 설치
uv sync
```

---

## 실행

```bash
# 기본 실행 (subset/split당 100개 수집)
uv run python -m src.main

# 수집 개수 지정 (subset/split당 N개)
uv run python -m src.main --max-samples 10

# 전체 수집 (주의: 4.4TB 규모)
uv run python -m src.main --max-samples 99999
```

### 파라미터

| 파라미터 | 기본값 | 설명 |
|----------|--------|------|
| `--max-samples` | 100 | subset/split 조합당 최대 수집 레코드 수 |

---

## 중단 후 재시작

파이프라인이 중단되더라도 `data/logs/checkpoint.json`에 완료된 subset/split이 기록됩니다.
동일 명령어로 재실행하면 완료된 조합은 자동으로 건너뛰고 이어서 진행합니다.

```bash
# 재시작 (완료된 subset/split 자동 skip)
uv run python -m src.main --max-samples 10
```

---

## 출력 파일

| 경로 | 설명 |
|------|------|
| `data/output.parquet` | 정규화된 전체 데이터 |
| `data/images/{subset}/{split}/` | 이미지 파일 (.jpg) |
| `data/logs/checkpoint.json` | 진행 상태 |
| `data/logs/run_log.jsonl` | 실행별 속도/시간 기록 |
| `data/failed/failed_records.jsonl` | 실패 레코드 (발생 시에만 생성) |
