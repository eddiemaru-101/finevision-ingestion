## 속도 개선 방법

### 현재 상태

- 단일 프로세스, 단일 스레드
- 실행 시 경고: `unauthenticated requests` → HF_TOKEN 미설정으로 rate limit 적용 중

### 방법별 예상 개선

| 방법 | 개선 배수 | 구현 난이도 | 설명 |
|------|----------|------------|------|
| HF_TOKEN 설정 | 1.5~2배 | 매우 낮음 | 환경변수 1줄. 인증된 요청으로 rate limit 완화 |
| ThreadPoolExecutor | 2~5배 | 낮음 | 이미지 저장 I/O만 스레드 병렬화. 코드 변경 최소 |
| 멀티프로세싱 | 8~16배 | 중간 | subset 단위로 프로세스 분할. CPU 코어 수에 비례 |
| 멀티프로세싱 + 스레드 | 30~80배 | 높음 | 위 두 방법 결합. 현실적 최대치 |
| 멀티머신 (40대+) | 585배+ | 매우 높음 | 1일 내 전체 수집 가능. 인프라 필요 |

### 1일 내 전체 수집 가능 여부

단일 머신으로는 현실적으로 불가능. 필요 속도: **281건/s (현재의 585배)**.

현실적 대안:
- 필요한 subset만 선택해서 수집
- 멀티프로세싱 적용 후 장기 운영 (최소 7~20일 예상)
