# FineVision 데이터 수집 파이프라인 — 설계 요약

## 개요

HuggingFace `HuggingFaceM4/FineVision` 데이터셋(185개 subset, 약 4.4TB)을 streaming 방식으로 수집하고,
공통 스키마로 정규화하여 Parquet 파일로 저장하는 파이프라인.

---

## 파일 구조

```
finevision-ingestion/
├── src/
│   ├── __init__.py
│   ├── explorer.py      # HuggingFace API로 subset/split 목록 동적 탐색
│   ├── downloader.py    # 이미지를 로컬 디렉토리에 저장
│   ├── normalizer.py    # N×M Flattening + 중복 제거 + 스키마 정규화
│   └── pipeline.py      # 전체 흐름 조합 + 배치/체크포인트/에러 처리
├── data/
│   ├── images/
│   │   └── {subset}/{split}/{record_id}_{index}.jpg
│   ├── logs/
│   │   ├── checkpoint.json
│   │   └── run_log.jsonl
│   ├── failed/
│   │   └── failed_records.jsonl
│   └── output.parquet
└── src/main.py          # CLI 진입점
```

---

## 처리 흐름

```
main.py
  └─ pipeline.run()
       ├─ explorer.get_all_subsets_and_splits()   → 185개 subset/split 목록
       ├─ load_dataset(streaming=True)             → 전체 로드 금지
       ├─ downloader.save_images()                 → ./data/images/{subset}/{split}/ 저장
       ├─ normalizer.normalize()                   → N×M 펼치기 + 중복 제거
       └─ ParquetWriter.write_table()              → 1000개 배치 단위 append
```

## 출력 스키마

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `record_id` | String | 원본 레코드 SHA256 식별자 (재시작 일관성 보장) |
| `uid` | String | 펼친 row 각각의 UUID |
| `subset` | String | FineVision subset 이름 |
| `split` | String | train / validation / test |
| `source` | String | 원본 데이터셋 출처 |
| `image_path` | String | 로컬 이미지 경로 (없으면 `""`) |
| `question` | String | `texts[n]["user"]` |
| `answer` | String | `texts[n]["assistant"]` |
| `metadata` | JSON String | 품질 점수 등 부가 정보 (Schema-on-Read) |

---

## 핵심 설계 결정

### N×M Flattening
레코드 1개에 이미지 N장, 질문-답변 쌍 M개가 있을 때 N×M개의 독립 row로 저장.

```
images=[img_0, img_1], texts=[qa_0, qa_1]  →  4개 row
```

### record_id (SHA256)
`uuid4` 사용 금지. `images[0] 바이트 + texts[0]["user"]` 조합을 SHA256 해시.
파이프라인 재시작 시 동일한 원본 → 동일한 ID 보장.

### Schema-on-Read
`metadata` 필드는 subset마다 구조가 다르므로 저장 시 JSON 문자열로 통째 보존.
`question`, `answer`, `image_path` 등 자주 쿼리할 핵심 필드만 개별 컬럼으로 분리.

### 중복 제거
`image_path + question` 조합 기준. 동일 조합이면 해당 row skip.

### 체크포인트
subset/split 완료마다 `data/logs/checkpoint.json` 갱신.
재시작 시 완료된 조합은 `load_dataset` 호출 자체를 건너뜀.

### 에러 격리
파싱/저장 실패 레코드는 `data/failed/failed_records.jsonl`에 기록 후 파이프라인 계속 진행.

---

## 제약 조건 준수

| 조건 | 처리 |
|------|------|
| `streaming=True` 필수 | `load_dataset(..., streaming=True)` 고정 |
| `trust_remote_code` 금지 | 코드 어디에도 사용 없음 |
| 이미지 없는 레코드 | `image_path=""` 로 저장 (skip 아님) |
| 중복 row | `image_path + question` 기준 skip |
| 에러 레코드 | `failed_records.jsonl` 기록 후 계속 진행 |
| `record_id` | SHA256 사용, uuid4 금지 |
